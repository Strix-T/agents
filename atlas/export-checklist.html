Atlas — Monthly Report Checklist

Set the manifest, export everything below into the client's _intake/ folder, then give Atlas the command.

STRIX
4 Golden Rules — read first
  1. Same window everywhere. Set every tool to the calendar month you're reporting (e.g. June 1–30) — never "last 28 days."
  2. Always Compare → previous month (or grab last month too). That's what creates the up/down arrows.
  3. Wait ~3 days after month-end to pull — Search Console runs a couple days behind.
  4. Screenshots count. Anything that won't export as CSV, just screenshot — Atlas reads images.

★ = essential · optional = nice-to-have · new = added this update · beta = on hold, skip for now

0. Set the manifest → _intake/manifest.yaml new

This is what tells Atlas which sections to build — do it first.

  • ★ client — the business name (e.g. "Spa 4109")
  • ★ plan — managed_hosting · essential_care · premium_growth (sets the report's scope)
  • ★ addons — e.g. [geo_grid]; leave [] if none. Only listed add-ons get built.
Scope follows the plan. Premium Growth = the full report. Essential Care = lighter (GSC + a short on-site summary). Managed Hosting = no recurring report. Atlas builds only what the manifest turns on — so you only need to grab the data for those sections.

1. Google Analytics (GA4) → _intake/ga4/

"What people did once they reached the website."

Open each report → set the month + turn on Compare (previous month) → Share/Export → Download CSV.

  • ★ Reports snapshot / Overview — visitors, new visitors, sessions, engagement
  • ★ Acquisition → Traffic acquisition — where visitors came from
  • ★ Engagement → Pages and screens — which pages they viewed
  • ★ Tech → Tech details — set the dropdown to Device category (phone/computer/tablet), not "Browser"
  • ★ Engagement → Events (Key events) — bookings, calls, form fills
  • ★ Visitors over time — last 6 months — export one report with the range set to the last 6 months, so the trend is real data (your other GA4 exports stay on the report month + compare)
  • optional Engagement → Landing page — where people enter the site

2. Search Console (GSC) → _intake/gsc/

"What people typed into Google, and whether you showed up."

  • ★ Performance → Search results. Toggle ON all four: Clicks, Impressions, CTR, Average position. Set range to Last 6 months, then Export → CSV (a ZIP with Queries, Pages, Dates). Drop the whole thing in.
  • optional Page indexing — export if you want Atlas to flag pages Google is skipping.

3. Google Business Profile (GBP) → _intake/gbp/ [screenshots]

"How people find you on Google itself."

business.google.com → the location → Performance → set the month. Screenshot each — and repeat for last month so Atlas can show the change:

  • ★ Overview — total Business Profile interactions
  • ★ Calls tab — the number
  • ★ Bookings tab — the number ← key conversion
  • ★ Directions tab — the number
  • ★ Website clicks tab — the number
  • ★ Views & Searches — "people viewed your profile" + "searches showed your profile" (with top search terms)
  • ★ Reviews — total count, star rating, and screenshot recent reviews newest-first
Why every tab matters: the Overview only shows the combined total. Atlas needs Calls / Bookings / Directions / Website clicks broken out — that's the real "people are contacting you" story (on Spa 4109 it was 1,309 interactions the Overview alone hid).

4. Booking system / POS → _intake/booking.md

Bookings finish on the booking platform (Vagaro, Square, Mangomint, GlossGenius…), not the website — so GA4 can't see them. Add the real number (a screenshot or one line is fine):

  • ★ # of online bookings this month (and last month)
  • optional revenue and/or total appointments, if easy to see
  • optional where bookings came from, if the platform shows it
This closes the funnel honestly. Without it, "bookings" looks near-zero when it isn't — exactly what tripped up the first Spa 4109 draft.

5. Month context → _intake/notes.md

60 seconds here turns flat numbers into a story. Jot anything that happened:

  • Promotions or ads run? · New pages or services? · Website changes?
  • Closures / holidays / seasonality? · Customer complaints? · What does the owner most want to know?

6. Geo-Grid local rank → _intake/local-rank/ new

Add-on only — grab this only if the manifest lists geo_grid. Skip it otherwise.

In Local Falcon: run the scan for the client's keyword (e.g. "lynchburg day spa"), then on the Scan Report use Export and grab all of it:

  • ★ the scan-report CSV (the ranked business list + ARP / ATRP / SoLV)
  • ★ the data-points CSV (the per-grid-point ranks — this draws the heatmap)
  • optional the map-image / scan PDF (handy visual reference)
AI Visibility (SAIV) — beta, on hold. The ChatGPT / AI-Overviews scans are slow and unreliable right now (scans stuck "in progress" for hours in testing). Don't grab these for monthly reports yet. Geo-grid (the Google scan) is fast and reliable — that's the one to use.

7. One-time setup (do once per client)

  • ★ Confirm GA4 marks bookings / calls / form-submits as Key events — if not, have Neo wire it up (without it, the funnel's bottom is empty).
  • Tell Atlas the site's main actions (book online, call, contact form, gift cards).
  • Note which booking platform the client uses.
  • Make sure you have access to all three: GA4, Search Console, Business Profile.
Then run:  "Atlas, build [Month]'s report for [client] — exports are in [folder path]."
Atlas reads the plan + add-ons from manifest.yaml, so you don't need to state the tier.
